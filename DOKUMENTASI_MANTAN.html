Laporan komprehensif mengenai siklus hidup pengembangan perangkat lunak (SDLC) yang diterapkan pada portal arsip siswa mandiri berbasis PHP & Spreadsheet.

Analogi: Pentas Seni Sekolah & Gedung Pencakar Langit

Landasan Konseptual

Dalam era digital, kemampuan menganalisis proses pengembangan produk sangat penting. Membangun aplikasi MANTAN (Muara Arsip Nama & Tanda Anak-didik Ngetop) diibaratkan seperti membangun gedung pencakar langit. Anda tidak bisa langsung menyusun bata tanpa blueprint (desain), perhitungan beban (analisis), dan tim ahli yang bekerja secara terstruktur.

Kami juga menerapkan analogi pentas seni: melakukan pendataan kebutuhan (Analisis), menggambar denah panggung (Desain), latihan rutin (Pengembangan), gladi bersih (Pengujian), hari H (Implementasi), dan evaluasi (Pemeliharaan).

The SDLC Lifecycle

A

Perencanaan & Analisis Kebutuhan

Identifikasi Kebutuhan Pengguna

Memahami kebutuhan akhir: Siswa memerlukan akses mandiri ke data profil yang tersimpan di file Excel sekolah tanpa melalui proses manual yang lambat.

Analisis Kebutuhan (Fungsional)

Sistem harus mampu memvalidasi NIS & NISN, memuat file .xlsx melalui library, dan menampilkan profil lengkap beserta foto siswa.

Analisis Non-Fungsional:

Keamanan data melalui PHP Session, responsivitas antarmuka menggunakan Tailwind CSS, dan efisiensi memori pembacaan data.

B

Pengembangan (Development)

Pemrograman (Coding)

Proses teknis penulisan skrip PHP untuk memproses database spreadsheet dan logika session di index.php.

Pengujian Unit

Verifikasi fungsi-fungsi kecil, seperti pembersihan spasi pada input menggunakan fungsi trim().

// Logic: Mencari siswa di file Excel
$reader = IOFactory::createReader("Xlsx");
$spreadsheet = $reader->load('data_peserta_didik_2025.xlsx');
$data_didik = $sheet->toArray();
C

Pengujian (Testing)

Pengujian Integrasi

Memastikan integrasi antara UI Tailwind dan Backend PHP Spreadsheet berjalan selaras.

Pengujian Sistem

Verifikasi menyeluruh terhadap alur login hingga munculnya dashboard peserta didik.

UAT (Acceptance)

Uji coba pengguna akhir (siswa) untuk memastikan sistem mudah dipahami dan ngetop.

D

Implementasi

Pemasangan perangkat lunak ke server (Deployment) dan pelatihan pengguna agar bisa memakai aplikasi dengan efektif dan efisien.

Web Hosting / XAMPP Deployment
E

Pemeliharaan

Fase berkelanjutan yang mencakup perbaikan bug (bug fixing) dan penambahan fitur fungsionalitas di masa depan secara berkala.

Continuous Improvement

Pemilihan
Metodologi Agile

Meskipun SDLC di atas terlihat linier (Waterfall), pengembangan MANTAN menerapkan prinsip Agile. Kami memulai dengan fitur terkecil yang bisa dicicipi (Login & Baca Excel), kemudian melakukan iterasi untuk penyempurnaan desain.

Spiral Method

Cocok untuk identifikasi risiko keamanan data agar tidak gosong.

DevOps

Sinkronisasi tim pengembang untuk pembaruan fitur terus-menerus.
